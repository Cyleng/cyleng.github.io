An Introduction of HTTPS SSL and TLS

Chuanyu Leng
2021-06-27

So you probably would have noticed there’s a tiny lock icon before the URL(Uniform Resource Locator) when browsing the Internet. Maybe you’ve heard people or articles previously mentioned HTTPS, SSL, or TLS many times. But what do these abbreviations mean? Is there a relationship between them?

Besides getting the content you need, safety is the most concern. The transmission of data via HTTP(HyperText Transfer Protocol) is unencrypted or in other words, plaintext. A sniffer tool could intercept this unencrypted data which possesses a security concern.

Usually, IT engineers use this method to analyze, debug and monitor network connections. But it can be used for malicious purposes. Imagine you submitted your login credential on a website, which provides HTTP access using the free wifi at a place like a park, coffee bar, or restaurant. Such kind of private information could be intercepted by someone with a malicious purpose.

For addressing this vulnerability, SSL( Secure Socket Layer) was introduced. . It was developed by Netscape and works above the TCP layer, and based upon the HTTP standard to encrypt data via TCP transmission. So HTTP combines SSL/TCP, now we got HTTPS(HyperText Transfer Protocol Secure, also refers to HTTP over SSL or HTTP over TLS).

When SSL was updated to ver. 3.0, IETF(Internet Engineer Task Force) standardized it and added in several other mechanisms. In addition, they rename SSL to TLS(Transport Layer Security). So TLS also can be considered as SSL 3.1. Above is the history of HTTPS, SSL, TLS.

The UA(User-Agent), an Internet browser mostly, can trace the certificate chain up to Root CA(Certificate Authority), and it can be found in UA’s trust list. Then the website is considered safe. Otherwise, the certificate is issued by a CA, not in UA’s trust list, or self-issued, the website would be considered unsafe by UA. In this case, usually, the browser( IE, Chrome, Firefox, etc..) would give a warning page to remind users or prevent access to the website.

This is a brief introduction to HTTPS, SSL, and TLS. Some information contained in this article may not be completely accurate nor detailed, however, this makes no difference how important HTTPS is to the modern internet.
